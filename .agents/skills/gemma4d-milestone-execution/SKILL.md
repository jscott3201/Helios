# Gemma4D Milestone Execution

## Trigger

Use when the user asks to implement or verify a specific Gemma4D milestone.

## Workflow

1. Read the milestone file and linked spec sections.
2. Restate scope and non-goals in a short plan.
3. Convert tasks into small edits and tests.
4. Run the milestone verification commands.
5. Capture evidence in benchmark/test/report paths.
6. Stop at milestone boundary.

## Required output

```markdown
# Milestone result

## Status
complete | incomplete | blocked

## Files changed

## Commands run

## Acceptance checklist

## Evidence paths

## Blockers / next input
```
