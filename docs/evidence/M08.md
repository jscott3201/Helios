# M08 SSD Cold Prefix Cache Evidence

This file records the local M08 work completed on 2026-06-30.

## Status

M08 is implemented as a Rust SSD cold-prefix cache layer in `gemma4d-kv`. It persists inactive prefix blocks as versioned JSON envelopes with checksums, maintains a persistent SSD index with disk-byte LRU eviction, restores matching blocks before prefill, rejects corrupt and wrong-namespace blocks, records raw read/write byte accounting, and rejects mid-decode SSD fetch attempts before file reads.

## Implemented

- Added `PersistedKvManifest` and `PersistedLayerManifest` with manifest/file versioning and Gemma 4 layout metadata.
- Added SSD block writer/reader with block checksums, manifest validation, and persisted `index.json`.
- Added `SsdPrefixCache` with disk budget, LRU eviction, read/write/hit/failure/byte accounting, and `restore_before_prefill`.
- Added explicit `SsdRestorePhase::MidDecode` rejection to keep SSD fetch out of decode.
- Added M08 tests for SSD exact restore, wrong namespace rejection, corrupt block rejection, disk-budget eviction, manifest field coverage, and mid-decode rejection.
- Added `m08_ssd_benchmark` evidence runner for 1K/4K/8K/16K fixture restores.
- Added SSD accounting to the TUI cache provider DTO and cache page.
- Updated the KV cache manifest schema with M08 versioning and checksum fields.

## Caveat

The M08 benchmark is a deterministic fixture benchmark over JSON metadata/payloads, not a real-model tensor TTFT benchmark. It records cold-vs-warm timing and raw bytes for the implemented SSD path; it does not claim a latency win over real prefill until native tensor export/import is wired.
