# M11 OpenAI-Compatible Server Evidence

This file records the local M11 work completed on 2026-06-30.

## Status

M11 is implemented as a localhost-only OpenAI-compatible server stub plus live TUI HTTP provider. The server supports chat completions, streaming SSE chunks, models, adapters, health, metrics, request admission/memory guard errors, and stable read-only control endpoints for TUI attach. The TUI can attach to the local server, display live health/metrics/cache/adapters/benchmark state, and render streaming chat smoke status.

## Implemented

- Added `gemma4d serve` with default bind `127.0.0.1:8080` and non-local bind rejection.
- Added `gemma4d-server::http` with stdlib HTTP/1.1 listener, request parser, JSON responses, SSE streaming responses, metrics, control endpoints, and tests.
- Implemented `/v1/chat/completions` non-streaming and streaming for a deterministic stub backend.
- Implemented `/v1/models`, `/v1/adapters`, `POST /v1/adapters/load`, `POST /v1/adapters/unload`, `/health`, and `/metrics`.
- Added request guard errors for unsupported model/temperature, oversized context, memory guard rejection, missing/unloaded adapters, and remote/path adapter load rejection.
- Added TUI `HttpProvider`, `--server-url`, live dashboard/cache/adapters/benchmark snapshots, and live Chat page streaming status.
- Added M11 server smoke evidence runner and ignored raw output directory.
