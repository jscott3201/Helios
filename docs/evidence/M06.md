# M06 MTP Speculative Decoding Evidence

This file records the local M06 work completed on 2026-06-30.

## Status

M06 is implemented through the engine/TUI/FFI contract and fixture exactness gates, with native assistant execution intentionally fail-closed. The native FFI can load and validate a real Gemma 4 MTP assistant artifact shape, and the opt-in native target graph now materializes the last hidden state plus final full-attention and sliding-attention shared KV views. `gemma4_mtp_draft_block` still does not execute the assistant graph. That remaining gap is tracked in `docs/decisions/0002-m06-mtp-fail-closed-until-native-assistant-execution.md`.

## Implemented

- Added `gemma4d-engine` MTP speculative greedy primitives for draft, verify, accept, rollback, fallback, metrics, and auto-disable.
- Added exactness tests for block size 1 and block size 2 rollback against non-MTP greedy scripted fixtures.
- Added MTP auto-disable for low acceptance, active adapters, and compressed active KV in M06.
- Added an M06 fixture evidence runner: `cargo run -p gemma4d-engine --example mtp_fixture -- --out benchmarks/out/M06/mtp-fixture-report.json`.
- Added native C ABI and Rust FFI functions for drafter load/free, MTP draft block, and token verification.
- Added strict drafter manifest validation for `gemma4_unified_assistant` artifacts, including the 4-layer assistant config and safetensors tensor inventory.
- Exposed `native_last_hidden` through the raw C step result and Rust `StepResult` view type. The pointer is owned by the KV cache and is valid until reset/free or the next cache-advancing call.
- Materialized native target last-hidden state plus final full-attention and sliding-attention shared KV views in the opt-in native graph.
- Updated the TUI MTP provider payload and page so acceptance, rollback, and auto-disable state are visible.
