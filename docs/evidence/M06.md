# M06 MTP Speculative Decoding Evidence

This file records the local M06 work completed on 2026-06-30.

## Status

M06 is implemented through the engine/TUI/FFI contract and fixture exactness gates. The opt-in native graph now materializes the target last hidden state plus final full-attention and sliding-attention shared KV views, loads the real `gemma4_unified_assistant` artifact, and runs native assistant block-size 1 and 2 draft generation. Native end-to-end speculative verify/accept/rollback against the real target+assistant artifacts is still incomplete; that boundary is tracked in `docs/decisions/0002-m06-mtp-fail-closed-until-native-assistant-execution.md`.

## Implemented

- Added `gemma4d-engine` MTP speculative greedy primitives for draft, verify, accept, rollback, fallback, metrics, and auto-disable.
- Added exactness tests for block size 1 and block size 2 rollback against non-MTP greedy scripted fixtures.
- Added MTP auto-disable for low acceptance, active adapters, and compressed active KV in M06.
- Added an M06 fixture evidence runner: `cargo run -p gemma4d-engine --example mtp_fixture -- --out benchmarks/out/M06/mtp-fixture-report.json`.
- Added native C ABI and Rust FFI functions for drafter load/free, MTP draft block, and token verification.
- Added strict drafter manifest validation for `gemma4_unified_assistant` artifacts, including the 4-layer assistant config and safetensors tensor inventory.
- Exposed `native_last_hidden` through the raw C step result and Rust `StepResult` view type. The pointer is owned by the KV cache and is valid until reset/free or the next cache-advancing call.
- Materialized native target last-hidden state plus final full-attention and sliding-attention shared KV views in the opt-in native graph.
- Added native MTP assistant tensor loading and a block-size 1/2 forward path using target token embeddings, pre-projection, 4 assistant layers over shared KV views, greedy logits, and post-projected hidden state.
- Updated the safe Rust drafter wrapper to borrow the target handle for the drafter lifetime.
- Updated the TUI MTP provider payload and page so acceptance, rollback, and auto-disable state are visible.

## Caveat

The M06 fixture acceptance gates pass, strict assistant loading is validated, and the opt-in native graph now executes real assistant draft blocks. This is not yet proof of full real-model MTP exactness: native `gemma4_verify_tokens` currently advances the cache over supplied draft tokens and does not provide exact accept/rollback semantics for the native C ABI loop.
