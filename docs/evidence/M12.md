# M12 tiny16 Profiling and Release Gate Evidence

This file records the local M12 work completed on 2026-06-30.

## Status

M12 is implemented as a tiny16 release-gate evidence layer over the current local runtime. The release gate validates real Gemma 4 12B 4-bit target generation through the current C ABI / MLX-LM helper path at 1K/4K/8K/16K/32K, deterministic server admission behavior, MTP/cache/adapter/server fixture paths, TUI release walkthroughs, server survival after controlled TUI failure, fallback/disable paths, and known limitations.

## Implemented

- Fixed `references/configs/tiny16.toml` by consolidating duplicate `[tui]` tables so the TUI validator can parse the tiny16 profile.
- Added `gemma4d-tui release-walkthrough`, a non-interactive TUI workflow that validates config, records benchmark launch, refreshes dashboard/cache/adapters/chat/MTP state, profiles render latency, writes snapshots, and exports Markdown/JSON reports.
- Added `gemma4d-bench` M12 real-target matrix runner `m12_real_tiny16_matrix`.
- Added `gemma4d-bench` M12 control-plane release-gate runner `m12_release_gate`.
- Added ignored `benchmarks/out/M12/` raw evidence directory scaffold.
- Added committed M12 release report, compliance matrix, and release-readiness review.

## Caveat

M12 validates the current local release-gate slice with real target generation through the MLX-LM helper plus deterministic control-plane fixtures. Hand-written native Gemma 4 graph serving remains a known limitation and should require fresh tiny16 benchmark evidence before any native-graph performance claim.
