# M05 Ratatui Operator Console Evidence

This file records the local M05 work completed on 2026-06-30.

## Status

M05 adds the first usable `gemma4d-tui` operator console surface. The crate now has a Ratatui/Crossterm/Tokio CLI, terminal lifecycle guard, event/reducer navigation model, mock and file providers, offline config validation, benchmark launch records, deterministic snapshots, and tests. The TUI remains offline-only for this milestone and does not call MLX scheduler, native FFI, or runtime mutation paths.

## Implemented

- Added TUI dependencies pinned through `Cargo.lock`: Ratatui 0.30.2, Crossterm 0.29.0, Tokio 1.52.3, Tokio-util 0.7.18, Clap 4.6.1, Serde 1.0.228, TOML 0.9.12, tracing 0.1.44, and color-eyre 0.6.5.
- Implemented `AppState`, `Action`, reducer, page navigation, and keybindings for Tab, Shift-Tab, 1..9, `?`, `r`, `v`, `b`, `q`, Ctrl-C, and Esc.
- Implemented Dashboard, Config, Benchmarks, Logs, and Help pages.
- Added dependency-aware placeholders for Chat, Cache, Adapters, and MTP.
- Added `RuntimeProvider` with deterministic `MockProvider` and offline `FileProvider`.
- Added TOML config validation with an intentionally invalid tiny16 fixture.
- Added benchmark launch records that capture exact command, output directory, status, and report path without spawning model/runtime work.
- Added terminal lifecycle restoration tests for normal quit and controlled error paths.
- Added deterministic snapshot generation and render latency profiling.

## Render / Terminal Notes

- Render timing uses deterministic Ratatui `TestBackend` snapshots, not a live terminal frame clock.
- The command runner stdin/stdout were not TTYs, so `cargo run -p gemma4d-tui -- --provider mock` used the headless launch path and exited cleanly.
- Terminal restoration is covered by lifecycle tests for normal restore and controlled error restore.
- Interactive idle CPU was not sampled in this noninteractive environment; this is recorded as a caveat in `benchmarks/out/M05/tui-usability-report.md`.
