# M03 Greedy Text Inference Evidence

This file records the local M03 work completed on 2026-06-30.

## Status

M03 now has a real Gemma 4 12B MLX 4-bit greedy generation path behind the existing C ABI. The default implementation is helper-backed: the native C++ shim owns a Python MLX-LM helper process for strict generation loads, while Rust still interacts only with opaque FFI handles. Native C++/MLX execution is also available behind `GEMMA4D_USE_NATIVE_GRAPH=1`; it loads the 1324 text tensors and runs a BF16 full-recompute prefill/decode path. That native path is parity-clean for the deterministic 8-token `Hello` smoke sequence and now has refreshed native 1K/4K/8K prefill measurements.

User-supplied Google QAT docs changed the target-selection evidence. Decision record `docs/decisions/0001-gemma4-12b-qat-target.md` keeps the current downloaded `mlx-community/gemma-4-12B-it-4bit` artifact as the M03 baseline while making the loader QAT-ready and requiring fresh tiny16 benchmarks before switching defaults to a QAT MLX artifact.

## Implemented

- Fixed the native MLX CMake gate for the Homebrew MLX package, which exports the `mlx` target rather than `MLX::mlx`.
- Downloaded `mlx-community/gemma-4-12B-it-4bit` into ignored `artifacts/models/gemma-4-12B-it-4bit`.
- Added strict native model-path/artifact validation for non-smoke target loads.
- Added native manifest validation for strict loads: `config.json` shape checks plus safetensor-header inventory checks for 1324 language tensors, optional multimodal tensors, 332 quantized weight groups, and mixed 4/8-bit quantization overrides.
- Added opt-in native MLX model-state loading behind `GEMMA4D_USE_NATIVE_GRAPH=1`; it loads only `language_model.model.*` tensors and validates loaded counts against the manifest.
- Added an opt-in native C++/MLX full-recompute text forward path: quantized embedding, 48 decoder layers, RMSNorm, RoPE, MLX scaled-dot-product attention, tied lm head, softcap, and greedy argmax.
- Matched MLX-LM BF16 execution semantics in the native graph: BF16 model scalars for embedding scale and softcap, MLX `power` for GeGLU, and BF16 logits by default. `GEMMA4D_NATIVE_FORCE_FLOAT32=1` remains a diagnostic escape hatch.
- Added a line-oriented MLX-LM helper owned by the native shim for real Gemma 4 text-only prefill/decode.
- Kept smoke target handles available for default FFI tests without loading the full model.
- Added Rust FFI `StepResult`, `prefill`, and `decode_one` wrappers around the existing C ABI, including helper memory telemetry.
- Implemented deterministic Rust greedy argmax sampling with finite-logit validation and lowest-token tie breaking.
- Added `gemma4d generate` with real prompt tokenization, repeated-token benchmark input, JSON metrics output, and graceful missing-model-path failure.
- Added chunked helper prefill with a 2048-token chunk size and retained the ignored full-model chunked-prefill parity test as the native-graph guardrail.

## Remaining Gap

The default execution path is real and local, but it delegates the Gemma 4 text graph to MLX-LM through a helper process. The opt-in native path now loads the MLX tensor state and executes short and long-context BF16 full-recompute prefill/decode with deterministic short-prompt parity. Follow-up work item `019f1733-45f9-7051-b44d-ec380ac3a594` tracks deciding when to replace the helper-backed default with the native graph.

The helper uses `model_config={"model_type": "gemma4"}` and `strict=False` because the downloaded config reports `gemma4_unified` and includes vision tensors, while MLX-LM 0.31.3_2 provides `gemma4.py` / `gemma4_text.py` but no `gemma4_unified` module.

The native loader now parses per-module quantization overrides so QAT MLX artifacts with mixed 4/8-bit projections can be evaluated. No QAT artifact has been downloaded or benchmarked as the M03 default yet.

The latest parity trace resolved the native graph gap for the short deterministic sequence. The root cause was BF16 execution semantics: embedding scale and softcap needed BF16 model scalars, GeGLU needed MLX `power`/BF16 constants to match `nn.gelu_approx`, and final logits needed BF16 casting. After those fixes, the prefix `[9259,236772,236772]` is bit-exact against MLX-LM through all hidden states and logits, and the native `Hello` run matches the helper sequence.
