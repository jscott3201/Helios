# M04 Reference Parity Harness Evidence

This file records the local M04 work completed on 2026-06-30.

## Status

M04 adds a repeatable reference-parity harness in `gemma4d-bench`. The harness reads a prompt corpus, runs the `gemma4d` candidate command, runs configured references, writes raw JSONL records, computes readable token diffs, and generates a Markdown report from raw output. The verified reference path is MLX Python via the existing `native/gemma4_mlx/scripts/gemma4d_mlx_lm_helper.py` line protocol. A llama.cpp-style command baseline can be configured with `--llama-cmd`; if its output cannot be parsed as generated-token JSON, the comparison is labelled `inconclusive`.

## Implemented

- Added prompt corpus `benchmarks/prompts/M04-corpus.tsv` with `hello_smoke` and `hello_divergent_prefix`.
- Replaced the placeholder `gemma4d-bench` crate with:
  - `run` command for candidate/reference execution.
  - `report` command for Markdown generation from JSONL records.
  - MLX Python helper reference integration.
  - Configurable command baseline support for llama.cpp/HF-style commands.
  - Token sequence diffing with passed/failed/inconclusive labels.
  - JSONL records containing exact commands, environment, model config/tokenizer hashes, generated tokens, metrics, stdout, and stderr.
- Added `benchmarks/out/M04/.gitkeep` and ignore rules so raw M04 outputs stay local while the directory placeholder is tracked.
- Generated `docs/evidence/M04-benchmark-report.md` from `benchmarks/out/M04/records.jsonl`.
