# M10 Dynamic Adapter Evidence

This file records the local M10 work completed on 2026-06-30.

## Status

M10 is implemented as a trusted local PEFT adapter registry and routing layer. The implementation imports PEFT adapter configs and safetensors weights, validates manifest compatibility, enforces trusted-path policy, supports load/unload/pin CLI controls, routes one active adapter per request, disables MTP while an adapter is active, partitions KV namespaces by adapter identity and weight hash, and exposes adapter summaries in the TUI.

## Implemented

- Added `gemma4d-adapters` manifest parsing, compatibility checks, trusted-path policy, PEFT importer, safetensors header validation, local registry persistence, route selection, and fixture output helper.
- Added `gemma4d adapter import|load|unload|pin|list` CLI commands.
- Added adapter-aware KV namespace and restore-rejection tests for RAM and SSD prefix caches.
- Added TUI adapter snapshot model, mock/file provider support, Adapters page rendering, and acceptance tests.
- Added deterministic `m10_adapter_fixture` evidence runner.
- Added M10 ignore rules so generated fixtures, registry JSON, and reports stay uncommitted.

## Caveat
